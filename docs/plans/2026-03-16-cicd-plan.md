## Task 2: Fix Libp2pKeyManagerTest — update stale multihash assertions

**Files:**
- Modify: `app/src/test/java/com/anyproto/anyfile/data/network/libp2p/Libp2pKeyManagerTest.kt:142-155`

**Background:** The test `derivePeerId produces correct multihash format` was written when SHA-256 multihash was used (`0x12` codec, 32-byte hash, total 34 bytes). The implementation was later updated to use **identity multihash** (`0x00` codec) as required by any-sync/libp2p for Ed25519 keys. The correct format (from `Libp2pKeyManager.derivePeerId`):
- `multihash[0] = 0x00` (identity codec)
- `multihash[1] = 0x24` (= 36, the length of the protobuf-encoded key)
- Total size = 38 bytes (2-byte header + 4-byte proto header + 32-byte raw key)

**Step 1: Run the failing test to confirm**

```bash
./gradlew test --tests "com.anyproto.anyfile.data.network.libp2p.Libp2pKeyManagerTest.derivePeerId produces correct multihash format" 2>&1 | grep -A5 "FAILED\|expected\|but was"
```

Expected: something like `expected:<34> but was:<38>` (size mismatch).

**Step 2: Update the 3 assertions in the test**

Find the test at line 142. Replace the current assertions:

```kotlin
// OLD (SHA-256 format — stale):
assertThat(peerId.multihash.size).isEqualTo(34) // 1 + 1 + 32
assertThat(peerId.multihash[0].toInt() and 0xFF).isEqualTo(0x12) // SHA-256 code
assertThat(peerId.multihash[1].toInt() and 0xFF).isEqualTo(32) // Hash length
```

Replace with:

```kotlin
// NEW (identity multihash format):
assertThat(peerId.multihash.size).isEqualTo(38) // 2-byte header + 4-byte proto header + 32-byte key
assertThat(peerId.multihash[0].toInt() and 0xFF).isEqualTo(0x00) // identity codec
assertThat(peerId.multihash[1].toInt() and 0xFF).isEqualTo(0x24) // 36 bytes (protobuf-encoded key length)
```

Also update the comment on the test itself (line ~151) from `// Multihash: <code><length><hash>` to:

```kotlin
// Identity multihash: <0x00 identity codec><0x24 = 36 length><4-byte proto header><32-byte raw key>
```

**Step 3: Run the test and verify it passes**

```bash
./gradlew test --tests "com.anyproto.anyfile.data.network.libp2p.Libp2pKeyManagerTest.derivePeerId produces correct multihash format"
```

Expected: `PASSED`

**Step 4: Commit**

```bash
git add app/src/test/java/com/anyproto/anyfile/data/network/libp2p/Libp2pKeyManagerTest.kt
git commit -m "fix(test): update multihash assertions to identity format (0x00) in Libp2pKeyManagerTest"
```

## Task 4: Create GitHub Actions CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Background:** No CI exists. Two jobs:
- `build` — unit tests + lint + APK; runs on all triggers; fast (~5 min)
- `emulator-tests` — instrumentation tests via `connectedAndroidTest`; runs on push to `main` and manual dispatch only; depends on `build` passing

**Step 1: Create the `.github/workflows/` directory and `ci.yml`**

```bash
mkdir -p .github/workflows
```

Create `.github/workflows/ci.yml` with this content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
  workflow_dispatch:

jobs:
  build:
    name: Build, unit tests & lint
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Cache Gradle
        uses: actions/cache@v4
        with:
          path: |
            ~/.gradle/caches
            ~/.gradle/wrapper
          key: ${{ runner.os }}-gradle-${{ hashFiles('**/*.gradle*', '**/gradle-wrapper.properties') }}
          restore-keys: ${{ runner.os }}-gradle-

      - name: Run unit tests
        run: ./gradlew test

      - name: Run lint
        run: ./gradlew lint

      - name: Build debug APK
        run: ./gradlew assembleDebug

      - name: Upload APK
        uses: actions/upload-artifact@v4
        with:
          name: app-debug
          path: app/build/outputs/apk/debug/app-debug.apk
          retention-days: 7

  emulator-tests:
    name: Instrumentation tests (emulator)
    runs-on: ubuntu-latest
    needs: build
    # Only run on push to main and manual triggers — not on every PR
    if: github.event_name == 'push' || github.event_name == 'workflow_dispatch'

    steps:
      - uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Cache Gradle
        uses: actions/cache@v4
        with:
          path: |
            ~/.gradle/caches
            ~/.gradle/wrapper
          key: ${{ runner.os }}-gradle-${{ hashFiles('**/*.gradle*', '**/gradle-wrapper.properties') }}
          restore-keys: ${{ runner.os }}-gradle-

      - name: Enable KVM
        run: |
          echo 'KERNEL=="kvm", GROUP="kvm", MODE="0666", OPTIONS+="static_node=kvm"' | sudo tee /etc/udev/rules.d/99-kvm4all.rules
          sudo udevadm control --reload-rules
          sudo udevadm trigger --name-match=kvm

      - name: Run instrumentation tests
        uses: reactivecircus/android-emulator-runner@v2
        with:
          api-level: 29
          arch: x86_64
          profile: Nexus 6
          script: ./gradlew connectedAndroidTest
```

**Step 2: Verify the YAML is valid**


Expected: `YAML valid`
